Root Cause of Final Failure (routines_feature)
- The `selfhost_kpsc_main_compiles_routines_feature` test was failing
  (got `12`, expected `9`) not due to a logic bug in the selfhost Pascal
  source, but because the selfhost binary cache was stale.
- After the Rust codegen fixes (VAR parameter save/restore exclusion,
  local aggregate save/restore exclusion), neither the binary cache nor
  the stage1 cache had been cleared.
- Clearing `/tmp/kpascal-selfhost-binary-cache/*` and
  `/tmp/kpascal-selfhost-stage1-cache/*` caused the binary to be rebuilt
  with the corrected Rust codegen, resolving the test.

Cache Management Note (updated)
- After any change to `src/codegen.rs` or other Rust source, BOTH caches
  must be cleared:
  - `/tmp/kpascal-selfhost-binary-cache/`  (stage1 Forth binary)
  - `/tmp/kpascal-selfhost-stage1-cache/`  (stage1 output per-input)
- The stage1 cache key is the selfhost Pascal source hash, not the Rust
  binary hash. A stale binary will regenerate when the binary cache is
  cleared, but already-cached stage1 outputs won't regenerate until the
  stage1 cache is also cleared.

What Changed In This Round
- Added multi-character string-literal AST/codegen support for expression
  position in selfhost.
- Added captured ordinal arithmetic evaluation for array/subrange bounds in
  selfhost type parsing.
- Added a lightweight selfhost const table so decl-time bound capture can
  resolve user-defined ordinal `const` names.
- Confirmed decl-time ordinal `const` capture across parenthesized arithmetic
  and `Ord/Chr(...)` forms used later by array/subrange bounds.
- Added an explicit ignored regression for leading-parenthesized subrange type
  syntax so the unresolved enum/subrange ambiguity stays visible without
  destabilizing stage1.
- Removed most normal-program special-case emit logic from
  `selfhost/compiler_codegen.inc`.
- Added real overlapping variant-record layout instead of pure flat field-count
  sizing.
- Generalized subrange bound capture beyond raw integer literals.
- Tightened real literal lowering from decimal reconstruction to
  `S" ... " READ-F32 DROP`.
- Extended the selfhost regression suite from the earlier 24-test baseline to
  the current 34-test stage1 suite.

Key Selfhost Delta
- `selfhost/compiler_codegen.inc`
  - `EmitProgram` no longer uses the old probe/direct-sample/captured-writeln
    fallback paths for normal programs.
  - Explicit seed generation was separated into `IsSeedCompilerProgram()` and
    `EmitSeedCompilerProgram()`.
  - `EmitAstRealLiteralText` now emits real literals through `READ-F32`,
    aligning literal parsing more closely with Rust `f32` parsing.
  - Set-range and low-bound-aware set emission remain in place.
- `selfhost/compiler_decl.inc`
  - record/variant capture now tracks offsets and shared union starts instead of
    treating variant cases as a pure flattened append-only list.
  - subrange capture now accepts major ordinal-constant paths:
    integer literals, enum consts, builtin consts, and existing subrange types.
  - captured array/subrange bounds now also evaluate arithmetic forms using
    `+`, `-`, `*`, `div`, and `mod` over supported ordinal constants.
  - `const` declarations now feed a lightweight ordinal const table used by
    captured declaration-bound evaluation.
- `selfhost/types.inc`
  - now stores selfhost user-defined const names/values for decl-time lookup.
- `selfhost/types.inc`
  - record size is now tracked as actual cell count via
    `ast_type_record_size`.
  - field type references are preserved via `ast_field_type_ref`.
- `selfhost/compiler_expr.inc`
  - continues to carry `AST_EXPR_REAL` and set-range AST support added in this
    stage1 pass.
  - now accepts multi-character `TOK_STR` in expression position as
    `AST_EXPR_STR`.
- `selfhost/compiler_lexer.inc`

  evaluation.

  backend/semantic parity should still be audited.

  - Additional review is still needed for broader real-operation parity, not
    only literal decoding.
